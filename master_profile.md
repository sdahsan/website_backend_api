### Project "Personalize AI Chatbot Assistant" – Portfolio AI Chatbot Assistant Product
* **Core Objective:** An interactive, conversational portfolio interface designed to serve as an intelligent digital representative for recruiters, stakeholders, and website visitors to explain Syed's professional background, client history, and technical capabilities.
* **Technical Stack & Operational Lifecycle:**
  * **Zero-Cost Dynamic Context Injection Pattern:** Bypasses expensive fine-tuning compute costs and layout re-training cycles entirely. The Node.js/Express backend uses file-system streams to read the local version-controlled `master_profile.md` knowledge file from the container root and dynamically injects it directly into the foundational `systemInstruction` parameters of the native Google Gemini API at runtime. Updates occur instantly via simple markdown edits and Git pushes.
  * **Stateful Multi-Tenant Session Management:** The backend application maps client requests to a secure session UUID (`session_id`). Transactional conversational lineage histories (User Prompt vs Model Response) and real-time accumulated token usage are written synchronously to a decoupled **Supabase PostgreSQL database**.
  * **Dynamic Token Interception & Lead Capture Locking:** Before relaying commands to the Gemini engine, the server aggregates token data. The exact moment a `session_id` surpasses a strict **500-token threshold**, the API halts execution, triggers a `paused_pending_auth` state, and forces a lead-generation modal on the React frontend. The UI completely locks user inputs until a verified **Name** and **Email Address** are captured.
  * **Slack Block Kit Authorization & Webhook Activation:** Submitted lead data fires an asynchronous HTTP POST webhook to a corporate Slack channel using an interactive **Block Kit administrative layout**. Admins review user metrics and click an **`[🔓 Approve & Resume Chat]`** option. Slack forwards a verified signature webhook back to the GCP gateway, updates the Supabase schema to `active`, and broadcasts a real-time event to the client browser to seamlessly unlock the UI text inputs.
  * **Automated Cloud Infrastructure (GCP CI/CD):** Code pushed to GitHub prompts a webhook to **Google Cloud Build**, which uses a multi-stage `Dockerfile` to build the app on a secure Alpine base layer (`node:20-alpine`). The immutable image registers inside the regional **Doha (`me-central1`) Artifact Registry** and auto-deploys to **Google Cloud Run** (`--min-instances 0`, `--max-instances 1`), allowing zero-cost idle times and instant scaling infrastructure.
